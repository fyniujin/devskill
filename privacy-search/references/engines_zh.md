# 国内可用搜索引擎指南

本文档说明 strict 模式下各引擎的国内可用性、隐私保护级别和自动降级策略。

## 引擎对比

| 引擎 | 国家 | 国内速度 | 隐私保护 | 数据收集 | 追踪 | strict 默认 | 备注 |
|------|------|---------|---------|---------|------|-----------|------|
| **SearXNG（本地）** | 本机 | ⭐⭐⭐⭐⭐ 极快 | 极高 | ❌ 无 | ❌ 无 | ✅ 最优先 | query 不出本机，最佳隐私 |
| **Yandex** | 俄罗斯 | ⭐⭐⭐⭐ 快 | 中等 | 部分 | 少量 | ✅ | 俄文界面可用，中文搜索能力较好 |
| **Startpage** | 德国 | ⭐⭐⭐ 中等 | 高 | ❌ 无 | ❌ 无 | ✅ | Google 代理，结果质量高 |
| **Qwant** | 法国 | ⭐⭐⭐ 中等 | 高 | ❌ 无 | ❌ 无 | ✅ | 欧洲隐私法保护 |
| **Brave** | 美国 | ⭐⭐ 较慢 | 高 | ❌ 无 | ❌ 无 | ✅ | 独立搜索引擎，不依赖 Google |
| **DuckDuckGo** | 美国 | ⭐ 不稳定 | 极高 | ❌ 无 | ❌ 无 | ⚠️ 最后 | 国内连接不稳定，仅作兜底 |

```
│              SearXNG（本地，最优先）                    │

└──────────────────────────────────────────────────────┘
```

## 推荐使用方式

### 最佳隐私（推荐）
```bash
python scripts/search.py "关键词" --privacy strict --engines searxng,yandex,startpage
```
- query 完全在本地
- Yandex + Startpage 作为离线后备

### 最佳速度
```bash
python scripts/search.py "关键词" --engines baidu,bing
```
- 国内引擎最快
- normal 模式下的推荐选择

### 平衡模式
```bash
python scripts/search.py "关键词" --privacy strict --engines yandex,searxng
```
- 速度隐私均衡
- Yandex 国内速度快 + SearXNG 极致隐私

## 常见问题：特定引擎故障

### Yandex 故障
- 可能原因：反爬升级、IP 限制
- 解决：等待自动降级到 Startpage/Qwant

### 全部 strict 引擎失败
- 可能原因：网络受限、DNS 污染
- 解决：
  ```bash
  1. 检查网络：ping yandex.com
  2. 配置代理（config.yaml 中 search.proxy）
  3. 降级使用 normal 模式
  ```

### DDG 在国内不可用
- 暂时现象，V1.2 规划增加 Smart DNS 方案
- 当前自动使用其他引擎替代

## 漏洞与限制

| 问题 | 影响 | 规避方式 |
|------|------|---------|
| 搜索引擎改版 | HTML 解析失败 | 引擎故障自动降级 |
| CDN 劫持 | 结果被篡改 | SimHash 校验 + HTTPS |
| DNS 污染 | 域名解析到错误 IP | DoH/DoT（规划中） |
| IP 过滤 | 特定 IP 被封锁 | VPN/代理 + 自动降级 |
