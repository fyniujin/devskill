# 企业微信智能机器人 API 参考

本文档汇总了企业微信智能机器人与语音消息相关的 API 能力，供开发者快速查阅。

---

## 一、智能机器人概述

当用户向智能机器人发送消息时，企业微信后台会向开发者配置的回调 URL 推送事件。开发者可以：

1. 接收消息回调
2. 被动回复消息（同步）
3. 流式回复消息（同步更新内容）
4. 主动发送消息（异步）

**官方文档**：https://developer.work.weixin.qq.com/document/path/101039

---

## 二、语音消息回调

当用户在单聊中向机器人发送语音消息时，回调格式如下：

```json
{
    "msgid": "CAIQrcjMjQYY/NGagIOAgAMg6PDc/w0=",
    "aibotid": "AIBOTID",
    "chattype": "single",
    "from": {"userid": "USERID"},
    "response_url": "RESPONSEURL",
    "msgtype": "voice",
    "voice": {
        "content": "这是语音转写后的文本内容"
    }
}
```

**核心要点**：企业微信已内置语音转文字功能，回调中的 `voice.content` 字段直接包含转写文本。**开发者无需自行接入 ASR 服务**。

---

## 三、消息类型支持

| 消息类型 | 接收 | 发送 | 说明 |
|---------|------|------|------|
| 文本消息 | ✅ | ✅ | 最常用 |
| 图片消息 | ✅ | ✅ | 临时素材 |
| 语音消息 | ✅ | ✅ | 接收时已转写为文字 |
| 视频消息 | ✅ | ✅ | 最大 100MB |
| 文件消息 | ✅ | ✅ | 最大 100MB |
| 图文混排 | ✅ | ❌ | 仅接收，群聊或单聊 |
| 模板卡片 | ❌ | ✅ | 富文本交互，被动回复 |

## 四、回复消息格式

### 4.1 被动回复（文本）

```json
{
    "msgtype": "text",
    "text": {
        "content": "这是回复内容"
    }
}
```

### 4.2 被动回复（语音）

需要先上传语音素材获取 media_id：

```json
{
    "msgtype": "voice",
    "voice": {
        "media_id": "上传后获得的media_id"
    }
}
```

### 4.3 流式回复

适合长文本生成场景，通过 `response_url` 分批次推送：

```json
{
    "msgtype": "stream",
    "stream": {
        "id": "STREAM_ID",
        "finish": false,
        "content": "..."
    }
}
```

## 五、安全机制

### 5.1 加密方式

- 所有回调消息使用 AES-256-CBC 加密
- IV 向量为 AESKey 前 16 字节
- 数据采用 PKCS#7 填充

### 5.2 权限要求

- 需要配置回调 URL、Token、EncodingAESKey
- 所有通信必须使用 HTTPS

### 5.3 安全限制

- 回调 URL 必须能处理 GET（验证）和 POST（回调）请求
- 每次回调的 msgid 需用于去重
- response_url 在消息接收后 48 小时内有效

---

## 六、频率限制

| 操作 | 限制 | 说明 |
|------|------|------|
| 被动回复 | 不限制 | 推荐用于实时交互 |
| API 调用 | 20次/分钟 | 主动调用企业微信 API |
| 主动发送 | 20次/分钟 | 异步发送消息 |
| 流式刷新 | 最长6分钟 | 用户发消息后 |
| 并发交互 | 3条/用户 | 同一用户同时消息数 |

---

## 七、关键注意事项

1. **语音转写由企业微信官方处理**，开发者无法控制 ASR 引擎选择
2. **语音消息仅限单聊**（`chattype: single`），群聊暂不支持语音
3. **response_url 有效期**：被动回复的 response_url 仅 48 小时内有效
4. **临时素材有效期**：上传的媒体文件（语音、图片）3 天后失效，需重新上传
5. **流式消息需主动结束**：发送 `finish: true` 结束流式消息

## 八、开发环境搭建

### 8.1 配置智能机器人

1. 登录企业微信管理后台
2. 应用管理 → 创建应用 → 配置智能机器人
3. 设置回调 URL/Token/EncodingAESKey
4. 开通接收消息权限

### 8.2 本地测试

使用内网穿透工具暴露本地服务（如 frp）：

```bash
frpc http -l 8080 -p your-domain.com
```

### 8.3 验证回调

企业微信发送 GET 请求验证 URL：

```
GET /callback?msg_signature=xxx&timestamp=xxx&nonce=xxx&echostr=xxx
```

返回解密后的 echostr 原文即可通过验证。

## 九、错误码参考

| 错误码 | 含义 | 处理建议 |
|-------|------|---------|
| 0 | 成功 | - |
| 40001 | 不合法的 secret | 检查应用凭证 |
| 40003 | 不合法的 UserID | 检查用户身份 |
| 40014 | 不合法的 access_token | 刷新 token |
| 60020 | 网络连接异常 | 检查企业微信服务器网络 |
| 84014 | 频率超限 | 降低发送频率，使用队列 |
| 45009 | 接口调用超过限制 | 实现退避重试机制 |

**完整错误码**：https://developer.work.weixin.qq.com/document/path/90313
